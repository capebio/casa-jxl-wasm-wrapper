# Security Policy

  ## Supported Versions

  We only provide security updates for the current main branch and the latest released 0.x version.

  ┌────────────────────┬────────────────────┐
  │ Version            │ Supported          │
  ├────────────────────┼────────────────────┤
  │ main               │ :white_check_mark: │
  │ latest 0.x release │ :white_check_mark: │
  │ older 0.x releases │ :x:                │
  └────────────────────┴────────────────────┘

  ## Reporting a Vulnerability

  Do not open a public issue for a security vulnerability.

  Use GitHub's private vulnerability reporting feature if available. If not, report the issue to the repository
  maintainers through a private channel only.

  Include:

  - affected version or commit
  - reproduction steps
  - impact
  - proof of concept, if safe to share
  - any suggested fix, if known

  What to expect:

  - Acknowledgment within 3 business days
  - Triage update within 7 business days
  - Fix or mitigation timeline depends on severity and release scope
  - If accepted, we will keep the report private until a fix is ready
  - If declined, we will explain why and close the report privately

  We may ask for more details during triage. Please keep the report confidential until we publish a fix or give
  permission to disclose.
