**Fix applied:** `process_dng_with_flags` implemented via `DngDecoded` struct + `decode_dng_raw()` + `process_dng_impl(output_flags, ...)`. `process_dng()` is now a thin wrapper passing all-outputs bitmask. Mirrors `process_orf_with_flags` exactly.

### G2 — `apply_look` always clones the buffer ✅ FIXED
**File:** lib.rs
**Fix applied:** Clone is now conditional: `if params.texture != 0.0 || params.clarity != 0.0 { let mut rgb16 = rgb16_src.to_vec(); ... } else { pipeline::process(rgb16_src, &params) }`. Common no-sharpening path avoids the full-resolution copy.

### G4 — ICC + sRGB both set in EncodeRgbaWithMetadata ✅ FIXED

**Fix applied:** `JxlColorEncodingSetToSRGB` is now called only in the `else` branch — only when no ICC profile is provided. When `icc_profile != nullptr && icc_size > 0`, only `JxlEncoderSetICCProfile` is called, eliminating the redundant/conflicting dual color encoding.

### G5 — ROI decode: full image decoded before crop

**Fix applied:** `finish()` now throws `CapabilityMissing("Progressive JXL encode requires a rebuilt WASM...")` when `options.progressive === true`. `previewFirst` and `chunked` remain documented no-ops (no WASM bridge support; no error thrown for those).
Verified: 2026-05-25

### G7 — Scheduler `drainQueue` silently swallows errors ✅ FIXED
**File:** scheduler.ts
**Fix applied:** `catch` block now calls `console.error("[jxl-scheduler] drainQueue error:", err)` and `setTimeout(() => this.drainQueue(), 50)` to log and retry after 50ms. Queued sessions are no longer permanently stuck on drain failures.
Verified: 2026-05-25

## 🟢 Confirmed Efficient Connections (no action needed)

These cross-layer connections are verified clean:

- **Backpressure chain**: jxl-stream → decode-session.push → scheduler.waitForDrain → decode-handler drain signal — fully connected, no double-gating.
- **Dedup boundary**: scheduler.DedupeRegistry owns fan-out; cache is content-agnostic — correctly separated.
- **Budget ownership**: session-level `stageStartMs` in decode-handler constructor; not per-stage; not in facade/cache/stream.
- **Preemption pause/resume**: scheduler sends `decode_pause`; decode-handler posts `decode_paused` ack; scheduler awaits ack before reassigning worker. Correctly synchronous WASM boundary respected.
- **Worker wiring**: one `onMessage` closure per worker lifetime (wiredWorkers WeakSet). No stale handler accumulation.
- **Zero-copy pixel transfer**: ArrayBuffer transferred via `postMessage([ab])`; facade reads it via HEAPU8.set; bridge writes directly to pre-alloc'd buffer. Full zero-copy chain.
- **Two-layer cache position**: jxl-cache sits beside the pipeline (not in it). Cache has no protocol knowledge. Caller (main.js/session layer) controls when to check/populate cache. ✅ Correct layer placement.
- **LookRenderer data flow**: lib.rs ProcessResult → take_rgb16_lb → JS → LookRenderer constructor (one-time transfer). Subsequent slider changes: only RGB8 crosses WASM boundary. ✅ Near-zero per-render transfer.
