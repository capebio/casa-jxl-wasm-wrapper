# Canonical Review Output Format

## Summary
2–4 sentence overall assessment. What the changes do, whether they look correct, dominant risk areas.

## Issues

### Issue N -- Severity: bug|suggestion|nit
- File: path/to/file.ext:LINE   (right side of diff for PRs)
- Description: precise explanation of the problem
- Suggestion: concrete fix or improvement

Severity values:
- bug: correctness, security, or breaking behavior issue
- suggestion: meaningful improvement
- nit: style, minor, or optional polish

Empty Issues section (or omitted) is valid and preferred when there is nothing to say.
