# Feature Design Notes

This folder contains per-feature design proposals created by Grok.

**Start here:** See [DESIGNS_INDEX.md](./DESIGNS_INDEX.md) for the complete list of notes, priority mapping, coverage of the REFERENCE_INDEX audit, and implementation status.

Each note is created after Grok aggregates and compares the relevant sections from the references (using REFERENCE_INDEX.md).

These notes are the primary source of truth for how a feature should be implemented in both WASM and Tauri.

The implementing agent should follow the main FEATURE_IMPLEMENTATION_TEMPLATE.md while using the specific design note for that feature as the main technical guide.

See the main template for the full required process (branching, benchmark wiring, cleanup/handoff, progress logging, etc.).

Follow-up blockers and broader unfinished business (including items still needing design notes) live in `ISSUES.md` in this folder. Every entry follows the Issue Entry Specification defined at the top of that file.
