JXL Codec Benchmark · Performance analysis

Decoder and encoder performance benchmark

Number of encode+decode cycles per configuration. Higher = more stable averages, less noise from JIT or thermal variation. 3 is enough for a quick comparison; 10+ reveals thermal throttling patterns. Each iteration is fully independent — no result caching.
Source files to include in each benchmark run. More files broaden coverage across image content (fine detail, flat areas, high noise) and reduce the impact of any single outlier on averages.
Quality
Quality (0–100 scale)
Controls perceptual fidelity. Maps internally to a Butteraugli distance target. 85 is visually indistinguishable from source for most photographic content. Above 90 gives diminishing returns at rapidly growing file sizes. JXL quality is not on the same scale as JPEG — JXL q=85 typically outperforms JPEG q=85 in both file size and visual fidelity.
Benchmark sizes:
Output pixel size (long edge)
Source images are proportionally resized before encoding. Full skips resizing and encodes at original resolution. Smaller sizes encode and decode much faster: 128–512 px is thumbnail range (~1–8 ms decode), 1920 px is display range (~30 ms), Full is archival range. Select multiple sizes to map the full size/latency curve.
Effort
Encode effort (1–9)
Controls how hard the encoder searches for a more compact representation. Higher effort = smaller files, much slower encode, identical decode speed and visual quality. Effort 3 suits real-time; effort 7+ is archival. Encode time increase 3→7 is roughly 5–10×, with 5–15% file-size reduction.
Encoder advanced
Modular — Switches from VarDCT (default, frequency-domain, photographic) to Modular mode (spatial, lossless-friendly). VarDCT gives better compression for photos; Modular excels at lossless and synthetic/non-photographic content.

Butteraugli target — Direct perceptual quality override. Lower = higher quality (0.5 = near-lossless, 1.0 = high, 3.0 = moderate). The Quality buttons map to this automatically — only override here for fine control.

Brotli effort — Compression level for JXL header and metadata (0–11). Affects header bytes only, not pixel data. Impact on total file size is small (<1%) for photographic content.
Codec toggles
Tier: detecting…
SIMD — Uses hardware vector instructions (SSE4.1, AVX2, NEON) for parallel pixel ops. ~2–4× decode speedup. Keep on for real-world measurements; disable only for scalar baseline.

Threading — Multi-core decode. Reduces latency for large images. Disable to get single-thread baseline.

Tier — Detected WASM capability (scalar / simd / simd-mt). Reflects the active build variant chosen by capability detection.

Progressive — Encodes with progressive scans so the decoder can emit coarse previews from partial data. ~1–2% file size overhead. Required for gallery and paint pages.

Preserve ICC — Passes the ICC color profile through decode. Disable for raw throughput measurement without color management overhead.

Metadata — Preserves Exif/XMP metadata. Not needed for speed testing.

Chunked — Splits encoded output into multiple Brotli-compressed chunks for HTTP streaming. Enables progressive delivery. Slight overhead vs. single buffer.

Lossless — Mathematically perfect reconstruction. Uses Modular mode internally. Files are larger; suited for text, diagrams, and archival.

Preview 1st — Inserts a low-resolution preview frame at the start of the stream so viewers can show something before the main decode completes.

Zero-copy buf — Skips buffer copy when pushing data into WASM. Faster on large inputs. Only safe when the source buffer is not reused after the push.
Decoder advanced
Downsampling — Decodes at a fraction of full resolution (÷2, ÷4, ÷8). Fast path for generating thumbnails from large JXLs without decoding every pixel. 1× = full resolution.

Architecture recommendations

Decoder selection by use case

Grid thumbnail (300px)

Decode with libjxl. Target: ~6 ms per image.

Pre-render at q=85 during ingest (~40 ms encode cost).

Lightbox preview (1080px)

Decode with libjxl. Target: ~25 ms per image.

Pre-render at q=95 during ingest (~35 ms encode cost).

Full resolution (2000px+)

Lossless encode + decode. jxl-oxide wins above ~3000px.

Use libjxl for lossy content, oxide for lossless.
