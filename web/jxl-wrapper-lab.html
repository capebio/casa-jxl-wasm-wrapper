JPEG XL wrapper and compare test lab

Compare JXL, WEBP, and JPEG performance in a drag-race track. Overlapping cards show relative paint positions.

Session worker routes JPEG XL through the browser session stack: context, scheduler, and worker messaging before the codec runs. Direct wrapper calls the wrapper facade straight against libjxl wasm with no session layer in between. Compare mode runs both paths on the same prepared thumbnail so you can see whether overhead comes from the session plumbing or the codec itself.

Batch tiles encode and decode resized thumbnails, not full-size source frames. The console shows both the original source dimensions and the smaller working size used for each tile, so thumbnail timings are not mistaken for full-resolution timings.
