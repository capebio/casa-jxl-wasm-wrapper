JPEG XL wrapper lab · Drag Race

JPEG XL wrapper test lab

Compare JXL, WEBP, and JPEG performance in a drag-race track. Race formats are exclusive per run so the result stays readable.

Session worker routes JPEG XL through the browser session stack: context, scheduler, and worker messaging before the codec runs. Direct wrapper calls the wrapper facade straight against libjxl wasm with no session layer in between.

Batch tiles encode and decode resized thumbnails, not full-size source frames. The console shows both the original source dimensions and the smaller working size used for each tile, so thumbnail timings are not mistaken for full-resolution timings.

Concurrency
Parallel worker threads processing images simultaneously. Match to your core count for maximum throughput. Lower values isolate sequential baseline latency.
Quality
JXL encode quality (50–100). Higher = better fidelity, larger file, slightly slower decode. 85–90 is visually lossless for photographic content.
Effort
Encode effort (1–9). Controls encoder search depth for compression efficiency. Higher = smaller files, much slower encode. Does not affect decode speed or visual quality.
Photon noise ISO
Synthetic photon noise target ISO. 0 = off. Try 800, 1600, 3200, or 6400 to add natural signal-dependent grain during encode.
Resampling
Encoder-native downsampling before JXL transform/coding. 1x = off; 2x, 4x, or 8x trade detail for smaller output and faster encode.
Upsampling mode (Phase 3)
0 (nearest) is non-negotiable for crisp pixel art. Combine with resampling>1. Warning badge in results for photographic misuse.
JPEG Reconstruction (Phase 3)
Fine control for JPEG→JXL fidelity + roundtrip quality. Best with actual JPEG sources (uses transcode paths).
Modular mode
Codec selection. Auto lets libjxl choose (VarDCT for photos, Modular for lossless). VarDCT (0) is optimal for photographs at lossy quality. Modular (1) suits lossless and line art.
Brotli effort
Brotli entropy effort for metadata boxes (−1–11). −1 = libjxl default. Higher compresses metadata more at encode-time cost. Rarely changes file size significantly for photographic content.
Advanced filters (new)
First-class promoted controls (Phase 1). See design note for details.
Group order (new)
Center mode + coords for better progressive/ROI behavior.
Buffering (Phase 2)
Rich tradeoffs (memory / density / latency). See design note for cjxl-derived guidance.
Expert (Phase 2)
Strong warning: for benchmarking/archival only. May increase size + artifacts. Mirrors cjxl --allow_expert_options.
Low Memory / Large Image (Phase 3)
Builds on Buffering (Phase 2). lowMemoryMode promotes to strategy=3 + streaming hints for minimal peak RAM. preferChunkedAPI signals the modern libvips/cjxl path on Tauri (JxlEncoderAddChunkedFrame + custom source; future slice). Browser already strong via streaming entrypoints. See production-chunked-paths.md design note for full rationale and future Tauri chunked source work.
Gain map (HDR / jhgm) (transport)
HDR gain map transport (ISO 21496-1 jhgm box). Supply a pre-encoded naked JXL codestream for the gain map image (the library wraps it using official JxlGainMapBundle APIs when the build supports <jxl/gain_map.h>). "Use demo" exercises the full encode/decode path with minimal placeholder bytes (content is not meaningful; file size delta and download badge prove the transport). Synergizes with LookRenderer tone-mapping workflows for real scientific HDR use. Graceful no-op on builds without symbols (see caps.gainMapEncode).
HDR signaling (Phase 3)
First-class HDR signaling (intensityTarget in nits, premultiply alpha for HDR, preferCICPForHDR policy). These ride advanced pairs for broad reach and are specially threaded on gain/animation/metadata paths. "HDR Info" badges appear in results when active. See hdr-signaling-color-priority.md (the gold-standard Phase 3 exemplar).
Decode speed tier
Decoder speed tier hint (0–4). Tells the encoder to structure the codestream for faster decoding. 0–1 = fully optimised, 2–3 = faster decode with small quality trade-off, 4 = fastest decode. Default 0 means no hint is applied.
Alpha distance
Per-channel encode distance for the alpha channel. Leave blank to inherit the main image distance. 0 = lossless alpha, 1 = same fidelity as main, 2 = more lossy alpha. Requires a rebuilt WASM binary with extra-channel encode support (caps.extraChannelEncode).

Extra-channel encode unavailable — rebuild WASM binary (see ISSUES.md §1/3).

Extra Channels demo (scoped)
Injects sample extraChannels with per-channel modular hints. See granular-extra-channel-modular.md. Full rich inspector is future.
Batch limit
Maximum images in one batch run. Increase to stress-test concurrency and memory pressure. Each image is fully independent through the pipeline.
Thumb size
Output pixel size (long edge) for each tile. Source images are resized before encode. 256–512 for gallery thumbnail testing; 1024+ for print/display benchmarking. Full size skips resizing.
Lossless
Mathematically perfect lossless encoding (no pixel loss). Uses JXL Modular mode internally. Files are larger than lossy at the same resolution. Best for text, line art, diagrams, and archival sources.
Compress boxes
Compress metadata boxes (ICC, EXIF, XMP) with Brotli inside the JXL container. Reduces file size when metadata is large. Has no effect on raw codestream output.
Force container
Always emit a JXL container (ISOBMFF wrapper) even when no metadata is present. Useful for compatibility with tools that require the container format. Ignored when Raw codestream is on.
Raw codestream
Emit a bare JXL codestream with no container wrapper and no metadata boxes. Smallest possible output. Overrides Force container. Not recommended for interchange — use only when you control both encoder and decoder.
JUMBF / C2PA (demo)
Attach JUMBF boxes (e.g. C2PA content credentials) as "jumb" container boxes. Uses the existing custom box infrastructure (no new FFI). The sample is a minimal illustrative stub — not a real signed credential. Decode reporting of JUMBF is future work; container roundtrip is reliable today. See docs/references/designs/jumbf-box-support.md.
Drag Race — Encodes the same image with JXL, WebP, and JPEG at matched quality, then races decodes side by side. Good for format comparison.

Session worker — Routes requests through the full browser pipeline (context → scheduler → worker messaging → codec). Measures realistic production latency including orchestration overhead.

Direct wrapper — Calls the WASM facade directly, bypassing all scheduling. Isolates pure codec speed with no orchestration cost. The delta between this and Session worker reveals pipeline overhead.
