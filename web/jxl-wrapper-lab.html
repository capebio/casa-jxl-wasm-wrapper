JPEG XL wrapper lab · Drag Race

JPEG XL wrapper test lab

Compare JXL, WEBP, and JPEG performance in a drag-race track. Race formats are exclusive per run so the result stays readable.

Session worker routes JPEG XL through the browser session stack: context, scheduler, and worker messaging before the codec runs. Direct wrapper calls the wrapper facade straight against libjxl wasm with no session layer in between.

Batch tiles encode and decode resized thumbnails, not full-size source frames. The console shows both the original source dimensions and the smaller working size used for each tile, so thumbnail timings are not mistaken for full-resolution timings.

Concurrency
Parallel worker threads processing images simultaneously. Match to your core count for maximum throughput. Lower values isolate sequential baseline latency.
Quality
JXL encode quality (50–100). Higher = better fidelity, larger file, slightly slower decode. 85–90 is visually lossless for photographic content.
Effort
Encode effort (1–9). Controls encoder search depth for compression efficiency. Higher = smaller files, much slower encode. Does not affect decode speed or visual quality.
Batch limit
Maximum images in one batch run. Increase to stress-test concurrency and memory pressure. Each image is fully independent through the pipeline.
Thumb size
Output pixel size (long edge) for each tile. Source images are resized before encode. 256–512 for gallery thumbnail testing; 1024+ for print/display benchmarking. Full size skips resizing.
Lossless
Mathematically perfect lossless encoding (no pixel loss). Uses JXL Modular mode internally. Files are larger than lossy at the same resolution. Best for text, line art, diagrams, and archival sources.
Drag Race — Encodes the same image with JXL, WebP, and JPEG at matched quality, then races decodes side by side. Good for format comparison.

Session worker — Routes requests through the full browser pipeline (context → scheduler → worker messaging → codec). Measures realistic production latency including orchestration overhead.

Direct wrapper — Calls the WASM facade directly, bypassing all scheduling. Isolates pure codec speed with no orchestration cost. The delta between this and Session worker reveals pipeline overhead.
