JXL vs JPEG vs WebP

Encodes the same source with JXL / JPEG / WebP at matched quality, then benchmarks real browser decode + reports file sizes. Click any size header in results to open the visual quality lightbox.

Quality tier
Matched quality applied to all three formats. Because JXL, JPEG, and WebP use different perceptual models, equal quality numbers do not produce equal visual results — JXL typically achieves better fidelity at the same or smaller file size. Use High (q=90) as the reference; compare file sizes in the results grid to see the efficiency gap.
JXL effort
JXL-specific encode effort (1–9). JPEG and WebP use their own fixed effort. Higher effort = smaller JXL files, much slower JXL encode. Use effort 7 to show JXL's maximum compression advantage; use effort 1 to benchmark encode speed. Does not affect decode time or visual quality.
Sizes to test
Output dimensions to benchmark. Each selected size triggers a separate encode+decode cycle for all three formats, with results side by side. 300 px is thumbnail range, 800 px is gallery card size, 1920 px is full display, Full uses the source at original resolution. Click any size row header in the results to open the quality lightbox.
